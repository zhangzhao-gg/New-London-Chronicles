# PRD: 新伦敦计划 (New London Project)

## 产品概述
一个基于冰汽时代美学的多人共享专注工具。用户通过完成番茄钟专注，为同一座城市"新伦敦"做贡献。城市永久记录每个用户的贡献。

**核心一句话**: 专注工作，建造新伦敦，城市会记得每一个人

---

## 技术栈
前端：Next.js + Tailwind CSS
后端：Next.js Route Handlers + 独立定时任务服务
数据库：Supabase
部署：自有服务器（Nginx 反代 + pm2 守护）+ 自有域名
定时任务：crontab 每分钟触发任务策略脚本

---

## 核心游戏循环

```
用户直接通过用户名登录 → 查看城市地图 → 选择任务(或被系统指派)
→ 用户自选专注时长 → 番茄钟开始 → 完成
→ 资源写入城市 → 城市记录该用户贡献 → 返回地图看到变化
```

## 页面结构
### 1. 登录页
- 输入用户名进行登录
- 新用户名自动创建账号；用户名已存在则直接登录


### 2. 城市地图页（主页面）
- 全屏冰汽时代风格背景图（AI生成，固定图片）
- 城市当前划分为6个可点击区块，未来持续扩展：
  -  **核心能量塔** 不可点击
  - ⛏️ **资源区**（散落的煤炭/散落的木箱/收集小屋...）
  - 🏠 **居住区**（帐篷/工人宿舍...）
  - 🏥 **医疗区**（医疗室...）
  - 🍲 **食物区**（猎人小屋/伙房...）
  - 🔭 **探索区**（灯塔/工作站...） 
  上述的后面三个点是指，未来会拓展建筑类型。
- 每个区块有SVG热区覆盖在背景图上
- **Hover效果**：气泡显示区块名称、当前状态、正在此处工作的人数
- **点击效果**：弹出区块详情面板
- 顶部设置栏：左侧是城市名、图案。中间是三个按钮显示物流（LOGISTICS）-当前选中、议会（COUNCIL）-未来可用，档案（ARCHIVES）-未来可用。 右侧是设置、语言选择（可选语音 中文/英文）、用户头像
- 顶部状态栏：显示城市当前资源总量（煤/木材/钢材/生食材/食物补给数量）、蒸汽核心（未来扩展）、温度（当前固定-20°C，未来每日更新一次）
- 左侧栏： （暂时不用，未来可扩展）
- 下边栏：包含2个按钮块。DISTRICTS显示城市大地图，FOCUS进入专注界面；若自动任务开启则直接分配任务进入专注页，若关闭则弹出任务选择界面。
- 右上方：有区域概览，记录当前在线用户数、健康状态（MVP固定显示"健康"，未来与疾病机制联动）、当前推进中的政策（MVP固定显示占位内容）
- 右下方：个人信息，包含头像名字

### 3. 区块详情面板（Modal）
- 点击具体区块后弹出
- 区块名称、描述、当前状态
- 正在此处工作的人数（如：「当前有3人正在工作」）
- 任务列表：每行显示任务名称、预计产出或效果、当前参与人数；每行末尾有独立操作按钮
  - 采集类/转化类 → **「前往工作」**
  - 建造类/工作类 → **「加入建造」**（显示剩余建造时长）
  - 原材料不足的转化类 → 按钮置灰，显示缺少的材料

### 4. 专注页面
- 冰汽时代风格插画
- 当前任务名称（无额外描述文字）
- 用户自选专注时长（输入分钟数）后开始倒计时，大字显示；有两个按钮：开始/暂停和重来（重来仅重置倒计时，不影响已写入的心跳贡献）
- 背景音控制：篝火，大雪，小雪分别对应focus,chill,rest
- 底层音乐播放器（lofi风格，可切换曲目）
- 番茄钟结束即工作结束，无放弃概念；用户可随时手动停止番茄钟返回地图，已写入的心跳贡献保留

### 5. 任务完成页
任务结算有三种触发：① 番茄钟自然结束 ② 用户手动停止 ③ 资源自然耗尽/建造完成。三种情况均进入此页面结算。

- 动画：资源入库效果
- 文字叙事：「{用户名}完成了{任务名}，为新伦敦贡献了{资源描述}」
- 若触发建造完成：「{建筑名}已建成，它将被命名为{参与者联名}的{建筑名}」
- 若用户开启了自动任务：系统直接安排下一个任务，跳转专注页
- 若未开启自动任务：提示用户选择下一个任务，或返回城市地图

### 6. 用户个人页 （暂不做）
- 用户名
- 累计专注时长
- 参与建造的建筑列表
- 历史贡献记录时间线

### 任务分类规则

**统一驱动原则：番茄钟是心跳节拍器，任务由资源状态驱动结束。**
番茄钟运行 → 每10分钟触发一次心跳 → 写入贡献/进度；番茄钟暂停 → 心跳暂停。
任务结束有三种触发：① 自然结束（资源耗尽/建造完成/原材料不足）② 番茄钟结束 ③ 用户手动停止 → 三种情况均立即结算贡献；资源点/建筑进度保留，下次可继续。
若用户暂停超过12小时，强制结束会话。

- **建造类**：城市建筑可无限建造（散落资源点等固定地物除外）。每次建造由算法创建实例，占据城市中一个预设槽位（见城市空间设定）。建造前预扣对应资源。多人可同时参与同一建筑实例，每10分钟心跳缩短10分钟剩余建造时长。进度归零时建筑立即建成，以所有参与者联名命名。
- **采集类**：资源点真实存在于地图中（如散落的煤堆），无需接任务，多人可同时采集同一资源点。每10分钟心跳从资源点采出N单位写入城市库存；资源点耗尽时立即结束并记录贡献。MVP阶段资源点数量设置极高（实际不耗尽），未来版本引入煤矿/伐木场等生产建筑作为正式资源来源。
- **转化类**：开始时检查城市库存原材料是否充足（不足则无法开始）。每10分钟心跳消耗N份原材料、产出M份成品写入城市库存；原材料耗尽时立即结束并记录贡献。允许多人同时参与。
- **工作类**：逻辑与建造类相同，多人可承接，每10分钟心跳推进任务进度，进度归零时任务完成。
- **探索类**：未来扩展，当前不做。

### 任务创建逻辑
所有任务实例均由后端算法决策创建，用户只能选择加入已存在的任务，无法自行发起。采集类与转化类是常驻动作，始终可用，无需创建实例。

后端每隔1分钟调用任务策略函数，算法如下：
1. 城市当前最缺乏的资源 → 加大对应采集动作的推荐权重
2. 资源充足时 → 创建建造类任务实例（如当前无进行中的帐篷建造，则创建一个）
3. 建造类任务所需资源不足时 → 回退，加大采集推荐权重

资源消耗：今日活跃用户（用户只要上线就要消耗1食物配给，若没有食物，则增加饥饿状态）数量每个人需要1份补给，每日能量塔需要消耗1000个煤炭，建造建筑需要消耗一些木材钢材等资源

---
### 系统自动指派逻辑
自动任务是用户的常驻开关设置。开启后，每次会话结算完毕系统自动安排下一个任务直接进入专注页，无需手动选择。关闭时，会话结算后提示用户自行选择。

自动指派优先级：
1. 城市当前最缺乏的资源 → 对应采集动作
2. 资源充足时 → 优先推送进行中的建造类任务
3. 建造类任务资源不足时 → 回退采集动作

## 城市空间设定

城市以能量塔为圆心，同心圆向外扩张，每圈为一个环。每个区块内预设若干槽位，各有固定地理坐标。建筑建成时自动占据所在区块最近的空槽。MVP阶段不做视觉表现，空间框架仅作为数据结构存在，供后续版本使用。

## 多人共享机制
- 无需实时WebSocket，使用轮询（每30秒刷新城市状态）
- 所有用户共享同一个城市数据，贡献实时累加
- 「当前X人正在工作」通过查询进行中的专注session得出
- 客户端每10分钟向服务端上报一次心跳，写入贡献进度；若用户断网或关闭页面，建造类/工作类任务进度保留，下次回来可继续；采集类无需继续，重新开始即可
- 建造类任务完成时，所有正在参与该建筑的用户通过轮询检测到状态变更，自动跳转任务完成页

## 城市日志系统
- 全局城市日志，所有人可见
- 每次会话结束（番茄钟结束、手动停止、或资源耗尽/建造完成）写入一条日志，记录本次会话总贡献
- 建筑完成也按 session 写日志，不额外合并成全局唯一建筑完成事件
- 格式：`[时间] {用户名} {贡献描述}`
- 示例：`[今天 14:32] 王五 在资源区采集了30单位煤炭`
- 示例：`[今天 09:15] 小明 建造了帐篷，它被命名为「小明&大壮的帐篷」`

## 视觉风格要求
- 色调：深棕黑（`#221810`）+ 暖橙（`#f4a462`）+ 琥珀（`#ff9d00`）
- 字体：`Newsreader` 衬线体，带工业/蒸汽朋克气质
- 背景图：AI生成，冰天雪地中的蒸汽朋克工业城市，固定不动
- 区块热区：半透明橙色描边，hover时高亮
- 整体氛围：压抑但有希望，寒冷但有炉火

## Demo范围（MVP）

**包含:**
- 用户注册/登录（用户名即可，新用户名自动创建账号）
- 城市地图 + 6个区块(包含能量塔，但能量塔点击没反应) + hover/点击交互
